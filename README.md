# Jai Chan OT Checker

ระบบช่วยทำฟอร์มเบิกค่าล่วงเวลา (OT) สำหรับ Jai Chan Spa — ทำงานในเบราว์เซอร์ทั้งหมด ไม่ต้องติดตั้ง ไม่ส่งข้อมูลออกเซิร์ฟเวอร์

**เปิดใช้งาน:** ดาวน์โหลด `index.html` แล้วเปิดด้วยเบราว์เซอร์ (Chrome/Edge แนะนำ)

**Repository:** [jaichanspa-admin/jai-chan-ot-checker](https://github.com/jaichanspa-admin/jai-chan-ot-checker)
**Live:** [https://jaichanspa-admin.github.io/jai-chan-ot-checker/](https://jaichanspa-admin.github.io/jai-chan-ot-checker/)

---

## ฟีเจอร์หลัก

- **นำเข้ารายงาน POS (.xlsx)** — ดึงบริการรอบสุดท้ายของวันแต่ละคนอัตโนมัติ
- **นำเข้าไฟล์กะ (.xlsx)** — จับคู่วันทำงานจาก roster รายคนรายวัน
- **คำนวณ OT อัตโนมัติ** — เทียบเวลาจบบริการกับเวลาสิ้นสุดกะ ปัดทุก 30 นาที
- **พนักงานชุดเก่า** — OT ขั้นต่ำ 1 ชม./วันทำงาน (มีตัวเลือก "ขอกลับก่อน")
- **แก้ไขก่อนปริ้น** — เพิ่ม/ลบ/แก้บรรทัดได้ก่อนออกฟอร์ม
- **ปริ้นฟอร์ม A4** — ปริ้นทีละคนหรือทุกคนในรอบเดียวกัน
- **Dashboard ประวัติ OT** — สรุปค่าใช้จ่ายรายพนักงาน รายรอบ กรองตามเดือนได้
- **ข้อมูลล่วงหน้า ส.ค.–ธ.ค. 2568** — ฝังประวัติ OT จากไฟล์เงินเดือนไว้ล่วงหน้าแล้ว

## วิธีใช้

### รอบปกติ (มีไฟล์ POS + ไฟล์กะ)

1. เปิด `index.html` ในเบราว์เซอร์
2. **Step 1** — ลากหรือคลิกอัปโหลด
   - รายงาน POS (sheet "ภาพรวม" — คอลัมน์ วันที่ / เวลา / รายการ / เวลาบริการ / พนักงาน)
   - ไฟล์กะ (sheet แยกรายคน ชื่อ `{รหัส} - {ชื่อ}`)
3. **Step 2** — ตั้งค่าเดือน รอบจ่าย (1 = 1–15, 2 = 16–สิ้นเดือน) และอัตรา OT (บาท/ชม.)
4. เลือกชื่อพนักงาน → ตรวจตาราง → แก้ไขได้
5. **Step 4** — ปริ้นหรือบันทึกเข้าประวัติ

### ดู Dashboard

กดแท็บ **📊 Dashboard** — ดูสรุปรวม รายพนักงาน รายรอบ กรองเดือนได้

## กฎ OT

| กรณี | วิธีคำนวณ |
|------|-----------|
| งานนวดเลยเวลาสิ้นกะ | `(เวลาจบ − เวลาสิ้นกะ)` ปัดลงทุก 30 นาที |
| พนักงานชุดเก่า | ได้ OT ขั้นต่ำ 1 ชม. ต่อวันทำงาน แม้ไม่เกินกะ |
| ขอกลับก่อน (ชุดเก่า) | ใช้เวลาจริง ไม่ได้ขั้นต่ำ |

**พนักงานชุดเก่า (รหัส):** 680602, 680603, 680606, 680609, 680613

**พนักงานที่ไม่มี OT (excluded):** 690626, 690622, 6901005, 6902049, 680617

## ฐานข้อมูลพนักงาน

ระบบมีฐานข้อมูลพนักงานในตัว (~30 คน) ใน `ROSTER` รวมชื่อเล่น ชื่อเต็ม และตำแหน่ง ไม่ต้องอัปโหลดไฟล์รายชื่อก็ได้ แต่สามารถอัปโหลดไฟล์ CSV/Excel เพื่ออัปเดตได้

## โครงสร้างไฟล์ localStorage

| Key | ค่า | คำอธิบาย |
|-----|-----|-----------|
| `jc_hist` | `[{period, label, y, m, half, empId, name, old, totalOT, workdays, rate, savedAt}]` | ประวัติ OT ทุกรอบ (flat records) |
| `jc_rate` | string | อัตรา OT บาท/ชม. |
| `jc_round` | `"floor"` / `"nearest"` | กฎปัดเศษ |
| `jc_map` | `{nickname: empId}` | การจับคู่ชื่อ POS ↔ รหัสพนักงาน |
| `jc_names` | `{empId: fullName}` | ชื่อเต็มที่แก้ไขแล้ว |

## ข้อมูลประวัติล่วงหน้า (ส.ค.–ธ.ค. 2568)

ไฟล์มี seed function ที่จะโหลดประวัติ OT จากไฟล์เงินเดือนปี 2025 เข้า localStorage อัตโนมัติในการเปิดครั้งแรก ครอบคลุม 9 รอบจ่าย (ส.ค. รอบ 2 ถึง ธ.ค. รอบ 2)

## ข้อกำหนดรูปแบบไฟล์นำเข้า

### POS Report
- Sheet ชื่อ "ภาพรวม" (หรือ sheet แรก)
- คอลัมน์บังคับ: `วันที่`, `เวลา`, `รายการ`
- คอลัมน์เสริม: `เวลาบริการ` (นาที), `รหัสการจอง`, `พนักงาน (1)`, `พนักงาน (2)`

### ไฟล์กะ
- Sheet "อธิบายรหัสกะ" (ถ้ามี) — คอลัมน์แรกคือรหัสกะ (เช่น `J10-19`)
- Sheet รายคน ชื่อ `{รหัสพนักงาน} - {ชื่อ}` เช่น `680602 - Fon`
  - Row 1: header
  - คอลัมน์ที่ 2: วันที่ (format `YYYYMMDD`)
  - คอลัมน์ที่ 3: รหัสกะ

## เทคโนโลยี

- **ไฟล์เดียว** — HTML + CSS + JS ทั้งหมดในไฟล์ `index.html`
- **SheetJS (xlsx 0.18.5)** — โหลดจาก CDN สำหรับอ่านไฟล์ Excel
- **localStorage** — เก็บข้อมูลในเครื่อง ไม่มีเซิร์ฟเวอร์
- **ไม่มี framework** — vanilla JS ล้วน

## ความเป็นส่วนตัว

- ไฟล์ POS, ไฟล์กะ และประวัติ OT ถูกประมวลผลในเบราว์เซอร์ของผู้ใช้
- ระบบไม่ส่งข้อมูลไปเซิร์ฟเวอร์ของ Jai Chan หรือบุคคลที่สาม
- ข้อมูลที่บันทึกไว้จะอยู่ใน `localStorage` ของเครื่อง/เบราว์เซอร์นั้นเท่านั้น
